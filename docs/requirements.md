# 留白code · 需求文档

> 用代码留白，为思考赋能

---

## 1. 项目概述

### 1.1 项目名称
**留白code** — 个人技术博客网站

### 1.2 项目定位
一个以 Java 后端技术、AI Agent 实践为核心的**个人技术博客**，同时作为面试作品展示平台，体现作者的技术深度与工程化能力。

### 1.3 目标用户
- **主要用户**：博主本人（撰写、管理文章）
- **访客**：面试官、技术同行、学习者

### 1.4 核心理念
- 极简设计，大量留白，阅读舒适
- 内容驱动，技术深度为上
- 以浅绿色为主色调，营造清爽自然的阅读氛围

## 2. 功能需求

### 2.1 前台（面向访客）

| 模块 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| **首页** | 最新文章展示 | 展示最新 5-8 篇文章摘要（标题、摘要、发布时间、标签） | P0 |
| | Hero 置顶区 | 展示最近一篇置顶/推荐的文章 | P0 |
| | 侧边栏个人信息 | 头像、简介、社交链接 | P0 |
| | 标签云 | 展示热门标签，点击可筛选 | P1 |
| | 站点统计 | 文章数、标签数、总访问量 | P2 |
| **文章列表** | 分页列表 | 按时间倒序展示所有文章，支持分页 | P0 |
| | 标签筛选 | 按标签筛选文章 | P1 |
| | 搜索 | 关键词搜索文章标题/内容 | P1 |
| **文章详情** | Markdown 渲染 | 支持代码高亮、表格、图片、数学公式 | P0 |
| | 目录导航 | 根据标题层级自动生成右侧目录，粘性定位跟随滚动 | P1 |
| | 上一篇/下一篇 | 文章底部导航 | P1 |
| | 文章阅读量 | 统计每篇文章的浏览次数 | P2 |
| **时间归档** | 时间轴视图 | 按年份和月份归档展示所有文章 | P1 |
| **关于我** | 个人介绍 | 技术栈、经历、联系方式、求职意向 | P0 |
| **RSS** | 订阅支持 | 输出 RSS/Atom 格式的订阅源 | P2 |

### 2.2 后台（面向博主）

| 模块 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| **文章管理** | 写文章 | Markdown 编辑器，支持实时预览、图片上传 | P0 |
| | 编辑文章 | 修改已发布的文章 | P0 |
| | 删除文章 | 软删除（放入回收站） | P0 |
| | 文章草稿 | 保存未发布的草稿 | P1 |
| | 标签管理 | 增删改标签，标签与文章关联 | P1 |
| **个人设置** | 基本信息 | 修改头像、简介、社交链接 | P1 |
| | 站点设置 | 站点标题、副标题、SEO 信息 | P2 |
| **数据统计** | 访问概览 | 总访问量、各文章访问排行 | P2 |

## 3. 非功能需求

| 类别 | 要求 | 说明 |
|------|------|------|
| **性能** | 首屏加载 < 2s | 前端按需加载、图片懒加载、CDN 加速 |
| **可用性** | 99.9% | 静态资源走 CDN，后端部署在云服务器 |
| **SEO** | 搜索引擎友好 | 服务端渲染（SSR）或预渲染，合理的 meta 标签 |
| **安全** | JWT 认证 | 后台接口鉴权，防止未授权访问 |
| **响应式** | 全端适配 | PC、平板、手机均能正常浏览 |
| **可维护性** | 代码规范 | 统一命名规范、Git 提交规范、API 规范 |

## 4. 技术选型

| 层次 | 技术 | 选型理由 |
|------|------|---------|
| **前端框架** | React 18 + TypeScript | 面试需求量大、类型安全、生态丰富 |
| **构建工具** | Vite | 快速 HMR，现代前端构建首选 |
| **路由** | React Router v6 | React 生态标准路由方案 |
| **状态管理** | React Query (TanStack Query) | 服务端状态管理，缓存/同步好帮手 |
| **UI 组件** | 自定义组件 + Tailwind CSS | 极简风格不需要重型 UI 库，Tailwind 快速且灵活 |
| **后端框架** | Spring Boot 3.x | Java 后端首选，师傅的主力 |
| **ORM** | MyBatis-Plus | 配合 MySQL，开发效率高 |
| **数据库** | MySQL 8.0 | 经典稳定，面试常考 |
| **缓存** | Redis | 缓存热门文章、会话管理等 |
| **认证** | JWT + Spring Security | 无状态认证，适合前后端分离 |
| **API 文档** | SpringDoc OpenAPI (Swagger) | 自动生成接口文档 |
| **部署** | 云服务器（前后端同机部署） | Nginx 反向代理 + 前端静态资源 |
| **CI/CD** | 可选：GitHub Actions | 自动化构建与部署 |

## 5. 页面结构

```
/
├── /                          # 首页
├── /articles                  # 文章列表页
├── /articles?tag=Java         # 按标签筛选
├── /articles/:id              # 文章详情页
├── /archive                   # 时间归档页
├── /about                     # 关于我页
├── /admin                     # 后台管理
│   ├── /admin/login           # 管理员登录
│   ├── /admin/articles        # 文章管理
│   ├── /admin/articles/new    # 写文章
│   ├── /admin/articles/:id/edit  # 编辑文章
│   ├── /admin/tags            # 标签管理
│   └── /admin/settings        # 站点设置
```

## 6. 数据库设计概览

### 核心表

| 表名 | 说明 | 主要字段 |
|------|------|---------|
| `article` | 文章表 | id, title, summary, content(md), content_html, cover_image, status(draft/published), category_id, view_count, created_at, updated_at |
| `tag` | 标签表 | id, name, color, created_at |
| `article_tag` | 文章-标签关联表 | article_id, tag_id |
| `site_setting` | 站点设置表 | id, key, value |
| `visitor_log` | 访问日志表（可选） | id, ip, page_url, user_agent, visited_at |

## 7. 优先级与里程碑

### Phase 1 — 基础版（MVP）
- 前台：首页、文章列表、文章详情
- 后台：文章 CRUD（含 Markdown 编辑器）
- 前台配色：浅绿色主题

### Phase 2 — 完善功能
- 标签筛选、搜索、归档页
- 相关文章推荐
- 评论区（Giscus 接入）
- 响应式适配

### Phase 3 — 锦上添花
- 数据统计
- RSS 订阅
- SEO 优化
- 访问统计
